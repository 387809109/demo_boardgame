Quick Start

Don't even think about using this module without reading the "How to Use This Module" file in the Help menu.

This module now automatically adjusts VPs! It provides many conveniences that take the place of manual piece movement and typing messages.

Playing Without ACTS (Automated Card Tracking Service)

  1. Choose a Scenario
    • 1517 Scenario (Turns 1-9)
    • 1532 Scenario (Turns 4-9)
    • Two Player Variant (Turns 1-9) Papacy vs Protestants
  2. Choose a Side
  3. Start Logging (File menu -> Begin Logfile)
  4. Deal Cards
    5. Using the menu in the main toolbar, deal cards to all players according to the setup in the Scenario Book.
  5. End Logging (File menu -> End Logfile)
    6. Send this logfile to the other players, or save it to a shared location (e.g., Dropbox).

Notes on PBEM and Logfiles

Players must use identical versions of the module. If even one player uses an outdated version, the game files will eventually become broken beyond repair and many module functions will not work properly. Never start using a newer module version in the midst of a game that was begun with an older version.

All players should be using different passwords. Don't leave the password blank. If you will be using the module on multiple computers, your password must be 100% identical on all computers or you will find yourself unable to access your side at some point.

Player sides are locked by a user's password. If a player needs to drop out of a game, try and get them to do a quick log or saved game in which they switch sides and become an "observer" before they depart the game. Otherwise a replacement player will be unable to take control of the side (it will still be locked to the old player and unavailable for selection).

Tips for avoiding logging difficulties:

  • Make 100% sure you step all the way through a logfile when playing it back.
  • If using the Load Continuation feature to load multiple logs sequentially, double and triple-check that you haven't missed any logs and have loaded them in the correct sequence, or your game will get board state mismatches with other players. VASSAL does not do any board state agreement checking!
  • Ideally, don't use Load Continuation at all. Review logs in isolation from each other: load a log, play it back, close it; load the next, play it back, close it; and so on. Only load the most recent log prior to starting a new one to record your play.

Playing With ACTS (Automated Card Tracking Service)

Follow the instructions above, but skip Step 4. The deck and all player hands will be maintained on the ACTS website, though you will still need to retrieve new ruler cards from the decks in VASSAL when needed.
